Тестовое задание для фронтенд-разработчика
Цель задания:
Оценить навыки кандидата в разработке на Vue 3 с использованием современных технологий, таких как Composition API, Pinia, Tailwind CSS, а также умение работать с API, реализовывать динамическую загрузку данных и создавать интуитивно понятный интерфейс. Функционал задания простой, загрузить данные с тестового сервиса jsonplaceholder и отобразить в виде таблички.

Технологии:
● Vue 3 (Только Composition API, <script setup>) TypeScript использовать нет необходимости.
● Pinia для сохранения и управления состояниями, кеширования данных.
● Tailwind CSS для стилизации.
● Radix-vue (shadcn-vue) по желанию (для UI компонентов).
● https://jsonplaceholder.typicode.com Бесплатный API для получения данных (постов и пользователей).


Используемые сущности и пример структуры данных:
Пользователи (Users)
● Эндпоинт: https://jsonplaceholder.typicode.com/users
Основные поля:
● {
  "name": "Leanne Graham",
  "username": "Bret",
  "email": "Sincere@april.biz",
  "address": {
    "street": "Kulas Light",
    "suite": "Apt. 556",
    "city": "Gwenborough"
  },
  "phone": "1-770-736-8031 x56442",
  "website": "hildegard.org",
  "company": {
    "name": "Romaguera-Crona"
  }
}

Посты (Posts)
● Эндпоинт: https://jsonplaceholder.typicode.com/posts
Поля:
● {
  "userId": 1,
  "id": 1,
  "title": "sunt aut facere repellat provident occaecati excepturi optio reprehenderit",
  "body": "quia et suscipit..."
}

Шапка приложения:
● Инпут фильтр:
○ Поле поиска по заголовку публикации (title).
○ Если поле пустое, запрашиваются все посты.
○ Пример запроса: https://jsonplaceholder.typicode.com/posts?title_like=sunt 
○ Если поле ввода пустое, допускается получение всех постов без пагинации, поскольку сортировка таблицы должна происходить по всем постам, а не только по видимым.
● Кнопка "Поиск". При нажатии отправляется запрос на API с учетом введенных ID альбомов.

Блок с таблицей:
● Максимальные размеры:
○ Высота: 600px, ширина: 600px.
○ расположена по центру, на любых экранах,
○ Если ряды таблицы превышает размеры, появляется внутренний вертикальный скролл.
● Динамическая загрузка:
○ Изначально отображается 30 строк.
○ При прокрутке до конца блока подгружаются следующие 30 строк в таблице.
● Отображение данных:
○ Все поля, полученные из API, отображаются в таблице как есть, в текстовом виде.
○ Отображаемые столбцы: 
■ Id - ID, 
■ title - Заголовок,
■ email автора (вместо userId) - Автор, 
■ body - Контент.
○ Если текст не помещается в ячейку, он обрезается (...) и отображается в подсказке (title).
● При клике в ячейку шапки таблицы:
○ сортируется таблица по всем данным, но отображаются снова только 30 шт. как при изначальном отображении.
Карточка пользователя:
● При клике на Email автора в таблице публикаций открывается модальное окно с карточкой пользователя.
● Брать данные из кэша приложения, полученного при первой загрузке.
● Обязательные поля к отображению в карточке:
○ • name —Имя
• username — Логин
• email — Электронная почта
• phone — Телефон
• website — Веб-сайт (кликабельная ссылка)
• company.name — Название компании
• address.street, address.suite, address.city — Адрес в одной строке
● Карточка должна содержать кнопку «Закрыть» и закрываться по клику вне области контента,  корректно обрабатывать случаи отсутствия полной или части данных.

	
Сценарий работы:
● При запуске приложения автоматически происходит поиск постов и пользователей и отображение постов в таблице.
● Отображается окно загрузчик (или скелетон, например рядов, ячеек или всей таблицы) в момент отправки запроса. Или заблокированная кнопка “Поиск” с иконкой анимацией загрузки.
● Автоматически отправляется запрос на получение всех постов.
● Отображается таблица с возможностью сортировки по столбцам.
Дополнительный функционал (необязательно, но желательно):
● Шапка таблицы прилипает к верхней части блока и видна всегда, даже при прокрутке.
● Переключение тем кнопка для переключения между светлой и темной темами.
● Сохранение Ид альбомов и темы в хранилище localStorage,
● Подсказки для текста, если текст не помещается в ячейку, он обрезается и отображается в подсказке при наведении (tooltip).
● Сохранять просмотренных пользователей в localStorage  (помечать в таблице публикаций email иным цветом).
● Приложение должно корректно обрабатывать недоступность API. При ошибке в ответах на запросы отображается уведомление: «Сервис временно недоступен. Пожалуйста, попробуйте позже.» alert, toast(shadcn vue, radix-vue)
